## §2. ★ AI 에이전트 철학 (최상위 원칙) ★

**v4.4 신규 섹션. 모든 시스템 설계의 근본 원칙.**

### §2.1 에이전트의 본질

**원래 돌아가는 시스템의 번거로움 해소가 본질.**

- 시스템이 먼저 존재
- 에이전트는 주희 대신 반복 관찰
- 주희는 알림만 확인

**에이전트의 역할:**
- 주희가 환율·유가 일일이 안 봐도 됨
- 음악 트렌드 변화 감지 시만 분석
- 건강 이상 시만 경고

### §2.2 트레일링 원칙

```
"에이전트는 트레일링이다.
 잘 만들어 놓기만 한다면 올려놓기만 하면 된다."
```

**의미:**
- 시스템 우선, 에이전트 나중
- 에이전트 없어도 시스템 작동
- 미리 에이전트용 공간 만들지 않음 (YAGNI)

**근거:**
- 에이전트는 "할 일"을 알아야 작동 → 시스템 정의 필요
- 에이전트는 "관찰 대상" 필요 → 데이터 축적 필요
- 에이전트는 "트리거 조건" 필요 → 정량 정의 필요

### §2.3 4가지 설계 습관

**모든 코드 작성 시 적용:**

1. **관찰 가능한 상태**
   - MongoDB 등 외부 저장
   - 함수 내부 변수만 쓰지 말 것
   - 에이전트가 읽을 수 있게

2. **구조화된 이벤트 로그**
   - `console.log` 대신 구조화된 로거
   - `{event, result, indicators}` 형식
   - 에이전트가 분석할 수 있게

3. **트리거 분리**
   - 실행 로직 ≠ 호출 방식
   - 같은 함수를 수동·정기·이벤트 다양하게 호출 가능
   - 에이전트가 다양한 방식으로 호출 가능

4. **결과 저장 및 재사용**
   - 결과 휘발 금지
   - 시계열 분석 가능하게
   - 에이전트 판단 자료 축적

### §2.4 허용 영역 vs 금지 영역

**허용: 관찰 + 알림**
- 매크로 지표 체크 (VIX/DXY/WTI, 이란전쟁 3조건)
- 건강 지표 모니터링
- 음악 트렌드 변화 감지
- 뉴스 긴급 신호
- **최종 판단은 항상 주희**

**금지: 행동 + 집행 (시기상조)**
- 자동 매매 주문 ❌
- 자동 손절·청산 ❌
- 자동 결제 ❌
- 근거: 복구 불가 영역

**Air Canada 판례 (2024):** AI 챗봇 오답 → 회사 법적 책임 + 배상금
**Gartner 2026:** AI 사망 사고 2,000건 예상 (자율 행동 에이전트 안전 실패)

### §2.5 판단 신뢰 Tier

**Tier 1-2 (위임 가능):**
- 숫자 비교
- 통계 계산
- 빈도 집계
- 매칭·필터링

**Tier 3-4 (위임 금지):**
- 의미 해석
- 가치 판단
- 전망·예측
- 맥락 이해

### §2.6 정량화 원칙

**모호한 판단 → 정량 지표로 환원**

```
❌ "트렌드가 변했다" (모호)
⭕ "특정 BPM 구간 비중이 30일 평균 대비 +30%" (정량)

❌ "삼성전자 매수해야 할 듯" (해석)
⭕ "120일선 위 + 거래량 평균 대비 1.5배" (정량)
```

**원칙:**
- 환원 가능 → 에이전트 위임
- 환원 불가 → 주희 영역 유지

### 작동 흐름

```
LibreChat 새 대화 → MongoDB conversations/messages 저장
         ↓
1시간마다 rag-autosave 자동 실행
         ↓
새 메시지 감지 (rag_autosave_state.last_run 이후)
         ↓
대화 텍스트 조립 → RAG API /embed
  (multipart: file + file_id + user + entity_id=AGENT_ID)
         ↓
files 컬렉션 INSERT → rag_file_ids 기록
         ↓
agents.tool_resources.file_search.file_ids 일괄 업데이트
         ↓
다음 질문부터 에이전트가 File Search로 entity_id 필터링 검색
```

## §10. ★ Phase 3 ④ 음악 트렌드 + Chordify + 레퍼런스 ★

### §10.1 음악 트렌드 감지 방법론 (v4.4 신규)

**주희 정립 - 기울기 기반 순위 변동 감지(Momentum-based Trend Detection)**

**핵심 지표:**
```
순위 결정 기준 = 기간당 조회수 변화량 (기울기) = Δviews / Δtime
```

**트리거 조건:**
- Top 50 신규 진입곡 3개 이상
- Top 10 신규 진입곡 1개 이상
- 동일 아티스트 복수 진입

**실무 설정:**
- 기간: 7일 이동평균 (요일 패턴 제거)
- 데이터: YouTube + Spotify + Reddit
- 필터: 주희 관심 장르(힙합+록)·BPM 범위(80-160)·키 분포

**학습 기간:** 초기 2개월 (데이터 축적만)
**검증 기간:** 3~6개월 (주희 피드백)

**분석 에이전트 출력:**
- 신규 진입곡 특성 (BPM, 키, 장르, 사운드)
- 해당 아티스트 배경·맥락
- 주희 현재 작업과의 정합성 평가
- 6~18개월 대중화 가능성 추정 (L2, L3 경로)
- Telegram 리포트
- MongoDB 저장 (검증용)

## §16. 매매 원칙 (2026.04.24 확정)

### §16.1 매매 2주 중단 결정

- 국장 포지션 전량 정리 (04.25) 후 2주 휴지

- 짧은 손절 = 엄마 각인 회로 재생 장치 = 악순환 원인

### §16.2 손절 원칙

- 1~2% 손절 = 착각 (ATR 내 노이즈 청산 → 연속 손절 → 심리 파괴)
- 정답: **구조 기반 5% 손절 + 포지션 역산** (감내 $손실 ÷ 손절% = 포지션 크기)
- Livermore "엄격한 손절" = 실행의 엄격함, 거리의 짧음 아님

### §16.3 재개 조건

- **깊은수면 80분 3일 연속**
- **잔고/차트 확인 충동 소멸**
- **자발적 대기 가능**
- 재개 시 국장 1종목 10만원 손절 5%부터, 손절 후 24h 쿨다운

### §16.4 미장 운용 원칙 (2026.04.24 확정)

1. **코인관련주(MSTR 등)**: 일봉 종가 120일선 위 + 거래량 확인 시 진입

   - 매크로 3조건 미충족 = 시드 5~10% 테스트
   - 충족 = 본진입
2. **기타 미장 스윙 중단** (24h 거래로 시초/종가 edge 없음)
3. **미장 단타** = 이벤트 직후 30분~1시간만
4. **일상 스윙 주전장** = 국장
5. **미장 시황** = 매크로 참고용
