**Knowledge Layer 개념 — 주희 시스템 핵심 공백 (카파시 LLM Wiki)**

배경: 안드레아 카파시 "LLM Wiki" 개념 — AI가 개인 데이터를 스스로 학습·가공하여 체계적 지식 저장소(Wiki)로 구축하는 시스템. 단순 RAG를 넘어선 능동적 지식 관리.

카파시 3계층 구조:
  레이어 1 (Raw Data): 원본 자료 보존
  레이어 2 (Knowledge Layer): AI가 가공·분류·링크한 위키 문서층 ★ 핵심 ★
  레이어 3 (Operation Layer): AI 작동 방식 및 규칙 문서화

주희 시스템 현재 상태:
  레이어 1 (Raw Data): ✅ 100% — MongoDB conversations 109개+, Phase 1 완성
  레이어 2 (Knowledge Layer): ❌ 20% — 가공 문서 없음, 교차 링크 없음 (가장 큰 공백)
  레이어 3 (Operation Layer): ⚠️ 40% — 마스터 컨텍스트 1개 + 에이전트 instructions 기본 설정

RAG vs Knowledge Layer 차이:
  RAG (현재): "도서관에서 그때그때 책 찾기" — 질문 시 의미 유사도로 검색 → 실시간 종합 → 관계 구조 없음
  Knowledge Layer (제안): "도서관에 목차·인덱스·교차 링크 미리 구축" — 사전 가공·분류·명시적 링크 → 질문 시 정리된 지식 즉시 활용

구축 시 변화 예시:
  현재: "코스피 인버스 전략" 질문 → 유사 대화 5~10개 반환 → Claude 종합 → 매번 다를 수 있음, 맥락 누락 가능
  구축 후: 사전 정리된 위키 문서 참조 → 일관된 답변 + 맥락 완결 + 관련 주제 자동 제시

주희 시스템 독자적 강점 (카파시 제안에 없는 것):
  1. 멀티 툴 통합 구조 (LibreChat + File Search + web_search + 도메인 tool 다층)
  2. 액션 레이어 (Phase 3 예정: OpenClaw + 자동매매 + Telegram 봇 → 실행 가능)
  3. 자동화 파이프라인 (1시간마다 자동 RAG 업데이트, Phase 1 완성)
